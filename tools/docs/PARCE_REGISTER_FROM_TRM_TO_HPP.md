# Генератор заголовочных файлов регистров AM335x из TRM (spruh73q.pdf)

Комплект Python-скриптов для автоматического парсинга Technical Reference Manual (TRM) процессора Texas Instruments AM335x и генерации готовых к использованию C/C++ заголовочных файлов с описанием регистров периферии.

## Обзор

Скрипты автоматизируют рутинную работу по переносу информации о регистрах из PDF-документации в код:

1. **parse_register.py** - парсит один конкретный регистр и генерирует `typedef union` с битовыми полями
2. **parse_registers_batch.py** - парсит множество регистров (по диапазону страниц или списку) и генерирует полный заголовочный файл модуля

**Возможности:**
- Парсинг координатного PDF (текст в HTML-формате) через PyMuPDF
- Автоматическое определение смещения, состояния сброса, битовых полей
- Извлечение полных описаний из таблиц "Field Descriptions"
- Объединение зарезервированных полей
- Идеальное форматирование вывода с выравниванием
- Сохранение оригинальных имён полей из TRM
- Автоматический расчёт `RESERVED` для выравнивания структуры модуля

# Скрипты
## parse_register.py
### Назначение: 
Парсинг одного регистра для быстрой проверки или генерации одиночного файла.  
### Использование:
```bash
python parse_register.py <PDF_файл> <имя_регистра>
```
## Пример:
```bash
# Парсинг регистра control_status
python parse_register.py spruh73q.pdf control_status

# Результат:
# [OK] Найден на странице 1466
# [INFO] Собрано 9 полей со страницы 1466
# [OK] Сохранено в: control_status_reg.h
```

Выходной файл: control_status_reg.h - содержит только typedef union для одного регистра. 

# parse_registers_batch.py
Назначение: Пакетный парсинг множества регистров и генерация полного заголовочного файла модуля со структурой и RESERVED полями.  
### Использование:  
### Режим 1: По диапазону страниц
```bash 
python parse_registers_batch.py --pages <начало>-<конец> --output <выходной_файл> --module-name <ИМЯ_МОДУЛЯ> --base-address <адрес> <PDF_файл>
```
### Режим 2: По списку регистров из файла
```bash
python parse_registers_batch.py  --list <файл_со_списком.txt> --output <выходной_файл> --module-name <ИМЯ_МОДУЛЯ> --base-address <адрес> <PDF_файл>
```

### Параметры

| Параметр | Описание | Пример |
|----------|----------|--------|
| `--pages` | Диапазон страниц PDF для парсинга всех регистров в нём | `--pages 1465-1476` |
| `--list` | Путь к текстовому файлу со списком регистров (по одному на строку) | `--list regs.txt` |
| `--output`, `-o` | Имя выходного заголовочного файла | `--output control_module.h` |
| `--module-name` | Имя модуля, используется для названия структуры `AM335x_<NAME>_Type` | `--module-name CONTROL` |
| `--base-address` | Базовый адрес модуля в шестнадцатеричном формате | `--base-address 0x44E10000` | 
### Примеры:
```bash
# Парсинг всех регистров Control Module (страницы 1465-1476)
python parse_registers_batch.py spruh73q.pdf --pages 1465-1476 --output control_module.h --module-name CONTROL --base-address 0x44E10000  

# Парсинг по списку регистров
python parse_registers_batch.py --list my_regs.txt --output my_module.h --module-name MYMODULE --base-address 0x48000000 spruh73q.pdf 
```

### Формат файла со списком регистров (regs.txt):
```text 
# Это комментарий - игнорируется
control_revision
control_hwinfo
control_sysconfig
control_status
control_emif_sdram_config
mosc_ctrl
deepsleep_ctrl
```

# Формат выходных файлов
## Структура генерируемого заголовочного файла
```c 
/**
 * @file CONTROL.h
 * @brief Register definitions for AM335x
 * @note Auto-generated by parse_registers_batch.py
 * @note Source: spruh73q.pdf (AM335x TRM)
 */

#ifndef _CONTROL_H_
#define _CONTROL_H_

#include <stdint.h>

#define __R     volatile const
#define __W     volatile
#define __RW    volatile

#ifdef __cplusplus
extern "C" {
#endif

/* (offset = 0x0) [reset state = 0x0] */
typedef union
{
    struct
    {
                                       /* Register control_revision */

        uint32_t    ip_rev_minor  : 6; // bits:0..5   (R) Minor Revision (Y).
        uint32_t    ip_rev_custom : 2; // bits:6..7   (R) Indicates a special version...
        // ...
    } b;
    uint32_t reg;
} control_revision_reg_t;

/* (offset = 0x4) [reset state = 0x0] */
typedef union
{
    // ...
} control_hwinfo_reg_t;

// ... ещё регистры ...

typedef struct
{
    __RW control_revision_reg_t          control_revision;              // (0000)
    __RW control_hwinfo_reg_t            control_hwinfo;                // (0004)
    __R  uint32_t                        RESERVED[2];                   // (0008..000C)
    __RW control_sysconfig_reg_t         control_sysconfig;             // (0010)
    __R  uint32_t                        RESERVED_2[11];                // (0014..003C)
    // ... ещё регистры и RESERVED ...
} AM335x_CONTROL_Type;

#define CONTROL_SIZE 0x0474

constexpr uint32_t CONTROL_BASE = 0x44E10000;
constexpr AM335x_CONTROL_Type * CONTROL = ((AM335x_CONTROL_Type *) CONTROL_BASE);

#ifdef __cplusplus
}
#endif

#endif /* _CONTROL_H_ */
```

## Особенности форматирования
1. #### Комментарий со смещением и сбросом перед каждым регистром:
```c 
/* (offset = 0x10) [reset state = 0x0] */
```
2. #### Битовые поля с полным описанием:
```c 
uint32_t    idlemode : 2; // bits:2..3  (R/W) Configure local target state management 00: Force Idle 01: No Idle 10: Smart Idle 11:
                          //                 Smart Idle wakeup capable
```
3. #### Структура модуля с RESERVED и смещениями:
```c 
__RW control_revision_reg_t          control_revision;              // (0000)
__R  uint32_t                        RESERVED[2];                   // (0008..000C)
```
4. #### Макрос размера структуры для проверки:
```c 
#define CONTROL_SIZE 0x0474
```

# Устранение проблем
## Проблема: "Регистр 'xxx' не найден"
### Причины:
- Опечатка в имени регистра   
- Регистр находится на других страницах  
- Регистр имеет другое имя в TRM  

### Решение:
1. Проверьте точное имя регистра в PDF  
2. Попробуйте поискать по части имени: grep -i "revision" spruh73q.txt (если есть текстовый экспорт)  
3. Увеличьте max_extra_pages в find_register() (по умолчанию 3)

## Проблема: "Сумма битов != 32"
### Причина: 
Парсер пропустил некоторые поля или неправильно определил границы.

### Решение:
1. Проверьте вывод скрипта - обычно показывает [WARN] Регистр 'xxx': сумма битов = XX (должно быть 32)  
2. Проверьте страницу в PDF - возможно есть скрытые зарезервированные поля  
3. Добавьте недостающие поля вручную или создайте issue  

## Проблема: Неправильные смещения в структуре
### Причина:   
Неправильный порядок регистров в PDF или пропущенные регистры.

### Решение:
1. Скрипт автоматически сортирует регистры по смещению  
2. Проверьте вывод - все смещения указаны в комментариях // (XXXX)  
3. Сравните с картой памяти в TRM (обычно в начале главы модуля)

## Проблема: "ModuleNotFoundError: No module named 'fitz'"
### Решение:
```bash 
pip3 install PyMuPDF
# или
pip install PyMuPDF
```
## Проблема: Краш при парсинге сложного регистра
### Решение:
1. Увеличьте max_extra_pages до 5  
2. Увеличьте размер буфера table_html = html[start_idx:start_idx + 50000]  
3. Проверьте, нет ли в PDF нестандартного форматирования для этого регистра  

## Дополнительные возможности
### Настройка форматирования
В parse_register.py можно изменить:

- #### Максимальную длину строки описания:
```python
if len(' '.join(cur + [w])) > 88:  # измените 88 на нужное значение
```
- #### Формат битового описания:
```python
bit_part = f"{bit_word}:{bit_range}"  # bits:0..7
# или
bit_part = f"{bit_word} {bit_range}"  # bits 0..7
```

3. Запустите пакетный парсер с --pages или создайте список регистров
